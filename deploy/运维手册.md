# MioOJ 部署与运维手册

本文档说明如何在云服务器上**部署、更新、暂停、重启** MioOJ，以及本地测试与生产的区别。

---

## 1. 架构速览

```
浏览器 → Nginx(:443/:80) → vibeoj(:8080) → MySQL(vibeoj 库)
                              ↓
                           web/ 静态前端
                           judge/ 判题
```

| 组件 | 作用 |
|------|------|
| **vibeoj** | C++ 后端：API + 静态文件 + 判题 |
| **MySQL** | 用户、题目、提交等数据（数据目录 `/var/lib/mysql/vibeoj/`） |
| **Nginx** | 反向代理 + HTTPS（对外 `https://rinr.top`） |
| **systemd** | 让 vibeoj 后台常驻、开机自启 |

---

## 2. 配置文件（本地 vs 生产）

| 文件 | 用途 | 提交 Git |
|------|------|----------|
| `deploy/config.example.env` | 配置模板 | 是 |
| `deploy/local.env` | 本地测试（端口 **8081**） | **否** |
| `deploy/production.env` | 生产环境（端口 **8080**） | **否** |

首次本地测试：

```bash
cp deploy/config.example.env deploy/local.env
# 编辑 local.env，填写 VIBEOJ_DB_PASSWORD，建议 VIBEOJ_PORT=8081
```

## 3. 日常开发流程

### 3.1 改代码 → 本地试跑

在项目根目录：

```bash
make clean && make && make run-local
```

- 访问：**http://127.0.0.1:8081**
- 按 **Ctrl+C** 停止
- **不影响** 正式站 `https://rinr.top`（8080 上的 systemd 服务）

> 注意：当前 `local.env` 与 `production.env` 默认连**同一 MySQL 库 `vibeoj`**，本地改的数据会出现在线上（仅端口不同）。若要隔离，另建 `vibeoj_dev` 并改 `local.env` 中的 `VIBEOJ_DB_NAME`。

### 3.2 改代码 → 推 Git → 上线

```bash
# 1. 本地 commit + push
git add .
git commit -m "你的说明"
git push

# 2. 在服务器项目目录执行（会自动停服 → 拉代码 → 编译 → 重启）
cd /home/user1/VibeCoding_oj
bash deploy/update-prod.sh
```

`update-prod.sh` 会自动：

1. 若 systemd 在跑 → **先 stop**（避免编译/重启时端口冲突）
2. `git pull`
3. `make clean && make`
4. **再 start** vibeoj

## 4. 生产环境：首次部署

只需做一次（需 sudo 密码）：

```bash
cd /home/user1/VibeCoding_oj

# 1. 确保 production.env 已配置（密码、DOMAIN 等）
# 2. 编译
make -j$(nproc)

# 3. 安装 Nginx 反代 + systemd 开机自启
sudo bash deploy/install.sh
```

完成后访问：**https://rinr.top**

HTTPS 若未配置，可执行：

```bash
sudo certbot --nginx -d rinr.top -d www.rinr.top
```

## 5. 生产服务：暂停 / 启动 / 重启

以下命令在**服务器**上执行，需 sudo。

### 5.1 暂停（停止网站，用户无法访问）

```bash
sudo systemctl stop vibeoj
```

- 8080 不再监听，Nginx 会报 502
- **MySQL 不受影响**，数据仍在

### 5.2 启动

```bash
sudo systemctl start vibeoj
```

### 5.3 重启（改配置或手动编译后常用）

```bash
sudo systemctl restart vibeoj
```

### 5.4 查看是否在跑

```bash
sudo systemctl status vibeoj
```

### 5.5 开机自启（install.sh 已配置）

```bash
sudo systemctl enable vibeoj   # 启用
sudo systemctl disable vibeoj  # 取消开机自启
```

## 6. 仅更新前端 JS/CSS（不重启后端）

若只改了 `web/` 下静态文件，**无需编译、无需 restart**：

```bash
git pull
# 浏览器 Ctrl+F5 强刷即可
```

---

## 7. 仅更新后端 C++（不 pull 代码）

已在服务器上改完代码时：

```bash
cd /home/user1/VibeCoding_oj
make -j$(nproc)
sudo systemctl restart vibeoj
```

## 8. 查看日志

### 8.1 应用日志（systemd）

```bash
# 实时跟踪
sudo journalctl -u vibeoj -f

# 最近 100 行
sudo journalctl -u vibeoj -n 100 --no-pager
```

### 8.2 文件日志（若 production.env 配置了 VIBEOJ_LOG_FILE）

```bash
tail -f /home/user1/VibeCoding_oj/logs/vibeoj.log
```

### 8.3 Nginx

```bash
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

## 9. Nginx 操作

```bash
# 测试配置语法
sudo nginx -t

# 改完配置后重载（不中断连接）
sudo systemctl reload nginx

# 停止 / 启动 Nginx（整站 HTTP/HTTPS 都会挂）
sudo systemctl stop nginx
sudo systemctl start nginx
```

Nginx 站点配置：`/etc/nginx/sites-available/vibeoj`  
模板来源：`deploy/nginx-vibeoj.conf.template` 或 Certbot 自动修改后的版本。

## 10. MySQL

```bash
# 服务状态
sudo systemctl status mysql

# 启动 / 停止
sudo systemctl start mysql
sudo systemctl stop mysql
```

数据文件位置：**`/var/lib/mysql/vibeoj/`**

导入迁移（有新 migrate_*.sql 时）：

```bash
mysql -u VibeOJUser -p vibeoj < server/db/migrate_phaseXX.sql
```

## 13. 常见问题

### Q: `make run-local` 报端口被占用？

8081 上可能还有上次未退出的进程：

```bash
ss -tlnp | grep 8081
# 找到 PID 后 kill，或 Ctrl+C 停掉之前的 make run-local
```

8080 被占用说明**生产 systemd 在跑**，不要用 `./vibeoj` 手动再起一个。

### Q: `update-prod.sh` 要输入 sudo 密码？

正常。重启 systemd 需要 sudo。若不想每次输密码，需自行配置 sudoers（不推荐把密码写进脚本）。

### Q: 换账号后编辑器里还有别人的代码？

已修复：草稿键为 `code_{用户ID}_{题目ID}`，存浏览器 localStorage，与账号绑定。

### Q: 数据库存在项目目录吗？

**不在。** 在 MySQL 数据目录 `/var/lib/mysql/vibeoj/`，与 `VibeCoding_oj/` 代码目录分离。
